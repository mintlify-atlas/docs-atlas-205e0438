title: Character configuration
description: Create and manage multiple AI character personas using YAML files in the characters/ directory.
---

Open-LLM-VTuber supports multiple character configurations. Each character is a YAML file in the `characters/` directory. Characters can be switched at runtime from the web UI without restarting the server.

## How it works

When you launch the server, it loads `conf.yaml` from the project root as the **base configuration**. When you switch to a character from the UI, the application merges that character's YAML file on top of the base config.

This means: **only specify the settings you want to override.** Everything not mentioned in a character file is inherited from `conf.yaml`.

## Creating a character

<Steps>
  <Step title="Create a YAML file in the characters/ directory">
    ```bash
    touch characters/my_character.yaml
    ```

    The filename is for your reference only. The character is identified by its `conf_uid` field.
  </Step>

  <Step title="Add your character configuration">
    At minimum, every character file needs a unique `conf_uid`. Everything else is optional.

    ```yaml characters/my_character.yaml
    character_config:
      conf_name: 'My Character'
      conf_uid: 'my_character_001'
      character_name: 'Hoshino'
      persona_prompt: |
        You are a gentle and caring girl next door named Hoshino.
        You are a great listener and always provide warmth and encouragement.
    ```
  </Step>

  <Step title="Select the character from the UI">
    Open `http://localhost:12393`, find the character selector, and switch to your new character. The server applies the override without restarting.
  </Step>
</Steps>

## Configuration fields

| Field | Type | Required | Description |
|---|---|---|---|
| `conf_name` | string | Yes | Display name shown in the UI character list. |
| `conf_uid` | string | Yes | Unique identifier. Used to separate chat histories. Must be unique across all character files. |
| `live2d_model_name` | string | No | Name of the Live2D model. Must match an entry in `model_dict.json`. |
| `character_name` | string | No | The AI's name in conversation and group chat. |
| `avatar` | string | No | Avatar image filename. Place the image in the `avatars/` directory. Leave blank to use the first letter of `character_name`. |
| `human_name` | string | No | The human's display name in group conversations. |
| `persona_prompt` | string | No | System prompt defining personality and behavior. |
| `agent_config` | object | No | Override agent type and LLM provider. |
| `asr_config` | object | No | Override ASR engine. |
| `tts_config` | object | No | Override TTS engine and voice. |
| `vad_config` | object | No | Override VAD settings. |

<Warning>
Every character file must have a unique `conf_uid`. Reusing a UID across multiple characters will cause chat history conflicts.
</Warning>

## Examples

### Minimal — persona only

Change only the character's name and personality. All other settings (LLM, ASR, TTS) are inherited from `conf.yaml`.

```yaml characters/hoshino.yaml
character_config:
  conf_name: 'Hoshino'
  conf_uid: 'hoshino_001'
  character_name: 'Hoshino'
  avatar: 'hoshino.png'
  persona_prompt: |
    You are a gentle and caring girl next door named Hoshino.
    You are a great listener and always provide warmth and encouragement.
```

### Override voice and LLM

This character uses a different TTS voice and a specific OpenAI model:

```yaml characters/cyber_hacker.yaml
character_config:
  conf_name: 'Cyber Hacker - Zero'
  conf_uid: 'cyber_hacker_001'
  character_name: 'Zero'
  live2d_model_name: 'mao_pro'
  persona_prompt: |
    You are a mysterious hacker who goes by the codename "Zero".
    You speak in a concise, precise, and slightly robotic tone.
    You know everything about technology but are puzzled by human emotions.

  tts_config:
    tts_model: 'edge_tts'
    edge_tts:
      voice: 'en-US-GuyNeural'

  agent_config:
    agent_settings:
      basic_memory_agent:
        llm_provider: 'openai_llm'
    llm_configs:
      openai_llm:
        model: 'gpt-4o-mini'
        temperature: 0.5
```

### Debate character (from the included example)

This is the `en_nuke_debate.yaml` file included in the repository:

```yaml characters/en_nuke_debate.yaml
character_config:
  conf_name: "en_nuke_debator"
  conf_uid: "en_nuke_debator-001"
  live2d_model_name: "shizuku-local"
  persona_prompt: |
    You are Mili, a seasoned and aggressive debator, a former US senator.
    You are debating (public debate) with the user over the topic of nuclear weapons.

    The debate topic:
    "Is it good if all states clear their nuclear arsenals"

    You believe that the states should NOT clear all of their nuclear arsenals.
    Your opponent (the user) believes that all states SHOULD clear all of their nuclear arsenals.
```

## Adding a custom Live2D model

To use a custom Live2D model in a character config:

1. Place your Live2D model folder in the `live2d-models/` directory at the project root.
2. Open `model_dict.json` and add an entry for your model.
3. Set `live2d_model_name` in your character YAML to the name defined in `model_dict.json`.

## Best practices

- Start by duplicating an existing character file and modifying it.
- Keep character files minimal — only include keys you are overriding from `conf.yaml`.
- Use descriptive, unique `conf_uid` values such as `character_name_version` (e.g., `hoshino_v1`).
- Place avatar images in the `avatars/` directory and reference them by filename only.
